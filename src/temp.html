GET /search?cat=<category_list>

This API returns searched result of the list of given category.

Category that to be searched would be defined with parameter "cat", and "comma (,)" would be used to seperate keywords if there is more than one.

Resource URL

http://<host>/api2/search

Parameters

name description type required
q search word string yes
cat category that to be searched. 
plural keywords would be seperated with comma. 		string yes
offset index number of the first entry in the result set (0-based, default value: 0) number no
limit maximum number of results of each category (default value: 10) number no
lang user interface language (ISO 639-2 Alpha-3 code) string no
input input device. kbd | voice (default value: kbd) string no
sort sorting method can be set in each category.
plural setting would be seperated with comma.
Syntax: sort=${category}:${sort_by}
ex) sort=tvshow:pop,movie:pop,channel:number

refer the table below for information of supported sorting method of each category 		string no
  • supported cat value
    Category Description
    vod VOD program
    movie movie VOD program
    tvshow TV SHOW VOD program
    vgroup VOD group category
    vseries VOD series
    channel TV channel information
    schedule TV program broadcasting schedule

★  If there is no value of "cat" parameter, it would work as unified search. If category that is not defined is stated, error (code: C0101) would be occured.
★  "offset," and "limit" would be applied to all stated catetories.

Additional Parameters

Parameters that only supported in specific categories.

name description type required supported category
m_field set field to search (title, genre, actor, director, tag). plural setting would be seperated with comma. string no vod, movie, tvshow, vgroup, vseries
p_rating set parental rating (0, 7, 12, 15, 19). plural setting would be seperated with comma. (e.g., p_rating=0,7 p_rating=0-7) string no vod, movie, tvshow, vgroup, vseries, schedule
vsort set overall sorting method of VOD search result (pop, new, abc).
ex) vsort=pop 		string no vod, movie, tvshow
filter set filter name of search result in each category. plural setting would be seperated with comma.
Syntax: filter=${category}:${filter_by}
ex) filter=vgroup:series 		string no vgroup
vsuppress exclude results that has specific condition out of the VOD search result (series). string no vod, movie, tvshow

★ "vsort" would be commonly applied in category "vod," "movie," and "tvshow".
★   If both "vsort," and "sort" are set, "sort" has a priority to be applied.
★ "vsuppress" would be commonly applied in category "vod," "movie," and "tvshow".

vsuppress
The option "vsuppress" can prevent the issue of VOD search result only includes contents of specific series. Mostly, "vseries" or "vgroup" category result would be used with it. 
filter
"filter" 옵션과 위의 "vsuppress" 옵션을 함께 사용해 시리즈물의 경우 VOD 검색 결과에서 제외하고 편성 category에는 시리즈물의 검색결과만 출력할 경우 사용할 수 있습니다.

ex) cat=vgroup,tvshow&filter=vgroup:series&vsuppress=series
  • Order of search result sorting method in each category
    Category Sorting by (${sort_by}) Description
    vod, tvshow, movie pops, default popularity inverse order -> chronological order (start date of license) -> alphabetical order

    episodes in the same series would be resorted in episode number inverse order.
      pop popularity inverse order -> chronological order (start date of license) -> alphabetical order
      new chronological order (start date of license) -> alphabetical order
    episodes in the same series would be resorted in episode number inverse order.
      abc, title alphabetical order
    schedule start, default chronological order of broadcating start time
      abc, title alphabetical order
    channel number-r, default chnnel number inverse order. e.g., 200, 100, 1
      number chnnel number order ex) 1, 100, 200
      abc, name alphabetical order
    vgroup count, default number of search result in a given category (organize) inverse order
      abc, name alphabetical order of category (organize)
    vseries count, default number of search result in a give series inverse order
      abc, name alphabetical order of name of the series

    Example of setting category's sorting method

  1. popularity order of "movie," and "tvshow". Title alphabetical order of "vgroup".
    1. vsort=pop&sort=vgroup:name
    2. sort=movie:pop,tvshow:pop,vgroup:name
  2. default order of "movie," "tvshow," "vgroup"
    1. parameters would not be appointed
    2. sort=movie:default,tvshow:default,vgroup:default
  • vsuppress supported value 
    Value Syntax Description
    series "vsuppress=series" or "vsuppress=series:limit" If the number of the same series in the search result is bigger than the limit, the series would not be included in the result. 
    Default value of limit is "1".
    If the limit is set as "0", all series contents would not be included no matter how many of them.
  • supported filter value in each category
    Category Filter name (${filter_by}) Description
    vgroup series only the organized category of VODs that belong to series would be included in search result

Fields

name description type
_param requested parameter list Map
hits additional informatio of search result Hit
${category} 요청한 각 카테고리의 검색 결과가 필드에 설정된다. CategoryHit

Hit Object Fields

name description type
total number of total search result number
data list of category id array of string

★  category order in data follows the "cat" parameter that is stated when it is requested.

CategoryHit Object Fields

name description type
total number of total search result of category number
data list of searched documnet (define search fields in each category) array of Document
